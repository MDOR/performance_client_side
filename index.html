Performance is User Experience

- Designing for Performance

Users expects the page to load in 2 seconds, after 3 seconds, up to 40% of users will abandon your site.

Your site's experience determines how much your audience trust your brand, return your site and share it with others.

CSS

CSS

CSS can produce a delay when the page is loading.

The browser will show content until the CSSOM (CSS Object Model) is ready.

CSSOM is basically a map of the CSS on the site, it contains inline styles and external resources.

CSSOM and DOM are combined and used to display pages by the browsers.

CSSOM is blocked by JavaScript.

The browser works unsing intervals of 16.66ms.

CSS Labels

Is possible to add media queries to manage the style to load in the page, for that reason we need to know if we have properly labels for our CSS files.

This method avoid load unnecessary code in the application and reduce the file's size. 

/*Syntax*/
<link rel="stylesheet" media="mediatype and|not|only (expressions)" href="path_to_file/file.css">

/*Best Practices*/
<link rel="stylesheet" href="common_styles.css">
<link rel="stylesheet" href="print_styles.css" media="print">
<link rel="stylesheet" href="portrait.css" media="orientation:portrait">
<link rel="stylesheet" href="min_resolution.css" media="(min-width: 40em)">

Concat CSS files

If the application will load external files, the HTTP request will increase the rendering time for this reason you should combine them in a single file in order to avoid a lot of HTTP requests.

They need to be ordered based on it's own precedence.

CSS Pipelines

The main areas to understand the CSS pipelines are:

  • JavaScript (JS)
  • Style (CSS)
  • Layout
  • Paint
  • Composite

Each one can be a blocker for rendering, these properties don't depend each other all the time.

CSS Flows

JS/CSS -> Style -> Layout -> Paint -> Composite
If the change modify the size or position, the browsers will check all the other elments for changes. The affected section into the page will be repainted and composite will be executed.
JS/CSS -> Style -> Paint -> Composite
If the change modify the color, background, shadow, etc. the element will be repainted.
JS/CSS -> Style -> Composite
If the change modify layer position the element will be repainted.

CSS execution

For each DOM modification (add or remove elements, add attributes, classes, animations) the browser will recalculate element styles in many cases specific section or the entire page (reflow).

Recomendations

We should reduce the changes into the DOM elements.

Avoid repaintings cycles longer than 16ms.

Use transform and opacity for animations.

Use will-change property if needed.

Use requestAnimationFrame.

Use flexbox instead other models.

Composing Components

The components clasification:

Atoms
The basic building blocks, basically our HTML tags. Examples: a, p, div, h1, navigation bar, etc.
Molecules
Combination of our different atoms in order to create elements, individually atoms aren't useful by themselves, combined they can respresent and doi something. Exaples: Search, Dropdowns, etc.
Organisms
Combination of different molecules, the reslut is block of content of the site. Examples: Header, Footer, etc.
Templates
At this moment the combination makes sense into design perspective, because we grouped different organisms to build a layout.

Steps

Find common modules into the page, like list, buttons, anchors, headings, etc. and find the common patterns into your site.

Check the component's states and variations.

Use the proper elments for your components.

Avoid specification.
